## What shipped

**Boilerplate + guardrails**

- Next.js 16 (App Router) + Tailwind **v4** — CSS-first `@theme`, no `tailwind.config.ts`.
- Strict 8pt grid, enforced by `eslint-plugin-tailwindcss` (`no-arbitrary-value`,
  `no-custom-classname`). The theme config alone cannot fail a build on an off-grid value —
  ESLint is the enforcement, and CI must run it or the grid is a suggestion.
- `CLAUDE.md` guardrails: brand tokens only, RSC by default, mandatory JSON-LD, mandatory
  `OptimizedImage`, RTL logical properties, lean serialized payloads.
- Shadow docs: `docs/index.md`, `docs/architecture.md`, `docs/ui-system.md`.

**Build-a-Box core** (logic complete, zero UI)

- `src/types/box.ts` — `BoxBuilderState` (rich, ephemeral) vs `BoxPayload` (lean, versioned).
- `src/lib/adapters/box-adapter.ts` — the serialization boundary. Pure functions.
- `src/store/box-builder.ts` — Zustand, capacity enforced, derived state in selectors only.
- **14 unit tests, all passing.** `node --test` on native TS — zero test dependencies.

**Hero + occasions**

- `ProductShowcase` — 4 flagship occasions, cream/green crossfade, hand-drawn line-art
  ornament field (candies, lollipops, twinkling stars).
- `OccasionsGrid` — all 11 occasions, each a real crawlable link.
- `src/lib/occasions.ts` — one taxonomy feeding hero, grid, sitemap, and JSON-LD.
- `SiteHeader` with a real mobile menu.

**SEO / AI SEO**

- Full metadata, `sitemap.xml`, `robots.txt`, `/llms.txt` — all generated from the taxonomy.
- JSON-LD: `Organization`, `WebSite`, `ItemList`.
- **robots.txt explicitly ALLOWS GPTBot, ClaudeBot, PerplexityBot, Google-Extended.**

## Bugs found (all real, all fixed)

1. **Fonts never rendered.** next/font vars were on `<body>`; Tailwind declares
   `--font-display` on `:root`. A custom property is substituted **where it is declared**, so
   it resolved to nothing and every heading silently fell back to the browser default.
   Days of design review happened with the typography switched off. Vars now on `<html>`.
2. **Tailwind compiled the banned classes.** v4 auto-scans the whole project, swept up
   `CLAUDE.md` and `docs/*.md` — which contain the *tables of banned classes* — and compiled
   `pl-4`, `mr-2`, `text-left` into the bundle. Fixed with `source(none)` + `@source "../../src"`.
   Bundle −15%.
3. **`next/image` rasterises SVG**, which kills SMIL silently. Needs `unoptimized`.
4. **Default SVGO config kills SMIL** — it minifies IDs and collapses groups, which `<animate>`
   references. Needed a custom config.
5. **`<a>` for internal nav** — full page reload on every click. Now `next/link`.
6. **`min-h-screen` on mobile** — `100vh` excludes browser chrome, so the hero was always
   taller than the visible area. Now `min-h-dvh`.
7. **`setState` in an effect** (React 19 lint) — replaced with `useSyncExternalStore`.

## Tried and reverted (do not re-attempt)

- **CSS-blurred foreground "depth stack."** Reads as dirt on the lens. A blurred circle has
  no silhouette, so the eye classifies it as a smudge, not an object. Occlusion needs real
  shapes — i.e. assets. Rejection is recorded in `ProductShowcase.tsx`.
- **`LazyMotion` + `m`.** Next preloads the dynamic feature chunk anyway; initial JS went
  236 → 244 KB gzip. Measured, lost, reverted.

**Blocking before launch**

   `localhost:3000`. **A canonical tag pointing at localhost de-indexes the site.** Highest
   stakes item on the list.
2. **All 11 occasion pages 404.** They are linked from the grid and listed in the sitemap.
   A sitemap full of 404s is actively harmful.
