## The decision that shaped the day

GitHub Pages serves **files**. There is no Node process. That killed the Server Action +
SQLite checkout built the day before (`b350430`) — not "degraded", *cannot execute*.

Chosen (with the user): **static export + WhatsApp ordering**, on the `github.io` subpath
for now, custom domain later.

**Consequence that matters most:** `pricing.ts` now runs in the customer's browser, which
they control, so its total is **no longer a guarantee**. That is survivable *only* because
the WhatsApp message carries the full itemised breakdown and a human at the shop confirms it
before money moves. **The shop is the enforcement point now.** The moment a payment gateway
is wired, this stops being true and pricing must go back server-side — `src/server/*` is
kept dormant and still tested for exactly that.

## Bugs found live (the useful part)

**1. Jekyll was serving the site, not our build.** First deploy failed at
`configure-pages` because Pages defaulted to "Deploy from a branch". GitHub then ran Jekyll
over the repo root and published a 1.7 kB stub — root `200`, every other route `404`.
Fixed with `enablement: true` so no manual settings change is needed.

**2. `TODO` placeholders were being fed to Google.** The `Store` schema shipped
`telephone: "TODO: +91-XXXXXXXXXX"`, `streetAddress: "TODO: shop no., street, landmark"`.
Cause: `isPlaceholder` was copy-pasted into the three *components* that render NAP — so the
visible footer correctly hid the unfilled fields — while `business.ts`, which **owns the
data and builds the schema**, had no such guard. A guard that does not live beside its data
gets forgotten by the next consumer. Moved it to `business.ts`; added a test.

**3. Every social link went nowhere.** `https://instagram.com/`, `facebook.com/`, `wa.me/` —
bare domains. Clicking "Instagram" in the footer took the visitor to *Instagram's homepage*,
on all 49 pages. `isPlaceholderSocial()` now tests the path; placeholders hide themselves.

**4. "Sign in" led to a blank page.** `/account/sign-in` and `/account/sign-up` used
`redirect()`, which does not work in a static export. They exported as empty shells — no
`h1`, no content, `robots: index, follow`, title identical to the homepage. Deleted; links
point at `/account`.

**5. CI failed while passing locally.** `vars.NEXT_PUBLIC_SITE_URL` is unset on the repo, and
**GitHub Actions passes an unset variable as an empty string, not `undefined`**. `??` only
guards null/undefined, so it accepted `""` → `new URL("")` threw → build died. Locally the
variable is genuinely *absent*, so `??` fired and it worked. `||` in all three readers.

## Built

- **`/shop`** (`ItemList` + `BreadcrumbList`) and **`/products/[slug]`** (`Product` + `Offer`,
  11 pages). These were the two biggest SEO surfaces and they did not exist. Slugs are
  **explicit, not derived from SKUs** — a SKU rename would 404 a URL Google had indexed.
- **`src/lib/whatsapp.ts`** — pure, tested message builders. Itemised down to each candy,
  because that breakdown *is* the safety mechanism (see above).
- **`offerSchema()`** — `shippingDetails`, `hasMerchantReturnPolicy`, `priceValidUntil`.
  Every value true and traceable to `pricing.ts` / `legal.ts`. Returns are
  `MerchantReturnNotPermitted` because made-to-order food genuinely cannot be returned.
- **`npm run preflight`** — one command answering "can this shop be announced yet?". Names
  what is *currently inert* because a value is unfilled. Exits 1 on a blocker; advisory in CI.
- **Domain switch is one env var.** Verified end-to-end: setting `NEXT_PUBLIC_SITE_URL`
  moved every canonical, emptied `basePath`, stripped `/al-hala` from every asset path,
  updated the sitemap and wrote the `CNAME`. No code changed.
- **Photos are drop-in.** `productImage()`; file into `public/products/`, set `imageUrl`.
  `Product.image` is **absent** until a real photo exists — it no longer falls back to the
  brand OG card, which would assert all eleven products look identical.

Removed: `src/app/order/[reference]`, the two auth routes, and five Next boilerplate SVGs
(`vercel.svg`, `next.svg`, …) that were shipping to production.

### 5 blockers, all needing real values

1. **`BUSINESS.whatsapp`** — nothing can be ordered until it is a real number.
2. **NAP** — street, postcode, phone. `Store` schema currently ships with no address, and a
   local business with no address cannot rank locally.
3. **`NAP_VERIFIED = false`** — geo and opening hours are withheld, because the defaults are
   guesses and wrong hours send a customer to a shut shop.
4. **Product photography** — Google renders **no** product rich result without a per-product
   image. All 11 `Offer`s are valid and doing nothing until this lands.
5. **Google Business Profile** (off-site) — the Maps pack outranks every blue link for
   "candy shop in Ratnagiri", and only a GBP feeds it.

### Warnings

- **Custom domain** — `github.io` is one domain shared by millions of repos and we are a
  *subdirectory* of it. Authority does not accrue. The biggest structural cap on ranking.
  User will purchase later; the switch is ready.
- Search Console not verified (env var wired, token needed) · socials unfilled (`sameAs`
  empty) · Maps URL unfilled.
